# 06 新規研究案: メモ外からのおすすめ

作成日: 2026-05-24

## 位置づけ

この資料は、既存メモの要約ではなく、新たに検討するための研究案リストです。腕や指の高さ変化を用いたカフレス血圧自己校正のような、すでに関心がありつつ一旦採用を見送っている案は主軸から外しています。

評価軸は次の4つを並列に扱います。

- 課題解決
- 研究としての評価
- 実用性
- 独自性

どれか1つが弱くても、アイデアとして面白ければ候補に残します。CHI 的な「問題の置き方を変える」研究も歓迎する前提です。

## 先に結論: 今のおすすめ

私が今いちばん推すのは、次の3方向です。

1. **測れない瞬間を研究対象にする**
   - ウェアラブルやIMU/PPGは「いつ正しく測れたか」より「いつ測れないか」のほうが日常利用では本質的です。
   - 課題解決、研究評価、実用性、独自性のバランスがよく、HCI と計測工学の両方で戦えます。

2. **不確かになった時だけ人に聞く human-in-the-loop センシング**
   - 常時ラベル付けではなく、モデルが迷った瞬間だけ micro-EMA 的に本人へ確認します。
   - 6軸IMU、スマホ、腕時計、M5Stack で小さく始められます。

3. **身体を測るのではなく、測定プロトコルをデザインする研究**
   - センサ精度を上げる研究ではなく、「人が自然に続けられる測定行為」を設計し、その結果としてデータ品質を上げます。
   - CHI らしい転換が狙えます。

## 案1: Signal Confidence Mirror

### 一言

生体値そのものではなく、「今の測定がどれくらい信頼できるか」をユーザに返すウェアラブル測定UI。

### 研究の転換

通常は `心拍を推定する`、`活動を分類する` が目的になります。ここでは目的を `推定値の信頼度をユーザと共有する` に変えます。

ウェアラブルは日常では、装着のずれ、手首の動き、接触圧、皮膚状態、汗、姿勢によって簡単に信号品質が崩れます。多くのシステムはそれを内部で捨てるだけですが、ユーザに「今は測れていない」「少し締めると改善する」「この姿勢では信頼度が低い」と返すと、測定が共同作業になります。

### 使うデータ

- IMU: 動作量、姿勢、周期運動、センサの安定性
- PPGがあれば: pulse amplitude、SNR、peak検出安定性
- M5Stack Core2: 6軸IMUだけでもプロトタイプ可能

### 最初の実験

1. M5Stack またはスマホを手首/腕/胸に装着する。
2. 静止、歩行、タイピング、腕振り、机作業、ポケット内などを測る。
3. 目標値の推定ではなく、`good / uncertain / invalid` の品質分類を作る。
4. ユーザに品質表示あり/なしで行動してもらい、データ有効率と負担感を比較する。

### 評価

| 軸 | 見込み |
| --- | --- |
| 課題解決 | 高: 日常計測の根本問題である信号品質に直接触れる |
| 研究評価 | 高: HCI、ウェアラブル、計測の境界にある |
| 実用性 | 中-高: 医療値を当てなくても価値が出る |
| 独自性 | 中: 信号品質推定は既存だが、ユーザとの協調UIに寄せると面白い |

### タイトル候補

- Signal Confidence Mirror: Making Wearable Measurement Uncertainty Visible to Users
- When the Sensor Hesitates: Designing Human-in-the-loop Confidence Feedback for Wearable Sensing

## 案2: Uncertainty-triggered micro-EMA

### 一言

センサが迷った瞬間だけ、ユーザに短い確認を求める活動・姿勢ラベリング手法。

### 研究の転換

既存の活動認識は、学習用ラベルが足りない、日常の活動が複雑、個人差が大きいという問題を抱えます。全てのラベルを集めるのではなく、モデルが不確かな瞬間だけ聞きます。

これは `人にたくさん質問する` 研究ではなく、`いつ質問すれば研究にもユーザにも得か` を探る研究です。

### 使うデータ

- 6軸IMU: 姿勢、動作遷移、停止/移動
- 心拍やPPGがあれば: 動作強度との不一致
- スマホ/PC: ユーザ応答UI

### 最初の実験

1. IMUで簡易活動分類を作る。
2. 分類確信度が低い、または活動遷移っぽい瞬間を検出する。
3. その時だけ `今なにをしていますか` を1問だけ聞く。
4. 一定間隔で聞く方式と比較する。

### 評価

| 軸 | 見込み |
| --- | --- |
| 課題解決 | 高: ラベル不足とユーザ負担を同時に扱う |
| 研究評価 | 高: CHI/UbiComp 的に通しやすい問い |
| 実用性 | 中: すぐ製品というより研究基盤 |
| 独自性 | 中-高: micro-EMA自体は既存だが、不確実性駆動に寄せると余地がある |

### 面白いポイント

ユーザの返答を「正解ラベル」としてだけ扱わず、`その瞬間に答えられる/答えられないこと自体` を文脈情報として扱えます。例えば、料理中、運転中、会話中、移動中では応答可能性が変わります。

### タイトル候補

- Ask Only When Unsure: Uncertainty-triggered Micro-EMA for In-the-wild Human Sensing
- Sensing the Right Moment to Ask: Active Labeling for Wearable Activity Recognition

## 案3: 測定不能性の地図

### 一言

日常生活のどの行動・姿勢・環境でウェアラブル測定が壊れるかを、ユーザごとに可視化する研究。

### 研究の転換

センサ研究は、測定できる条件で精度を上げがちです。しかし実用上は「いつ壊れるか」を知らないと使えません。そこで `測定可能な身体情報` ではなく `測定不能性` を対象にします。

### 実験案

- 手首装着、腕装着、胸部装着、ポケット内などを比較する。
- 行動: 歩く、座る、タイピング、食事、荷物を持つ、スマホ操作、寝転ぶ。
- 出力: 行動ごとの欠損率、信号品質、復帰時間、ユーザが感じる邪魔さ。

### 評価

| 軸 | 見込み |
| --- | --- |
| 課題解決 | 高: 現実利用で効く |
| 研究評価 | 中-高: 評価設計が良ければ強い |
| 実用性 | 高: デバイス設計・装着位置設計に直結 |
| 独自性 | 中: 方向性は地味だが、問いの置き方が良い |

### ひねり

`高精度な推定器` を作らない。むしろ `どの瞬間を推定してはいけないか` を明らかにする。これは安全性、説明可能性、ユーザ信頼に接続できます。

## 案4: 日常動作をキャリブレーション・イベントにする

### 一言

階段、立ち上がり、深呼吸、歩き始め、スマホを取るなど、日常に自然に出る動作をセンサの再調整イベントとして利用する。

### 研究の転換

キャリブレーションは普通、面倒な初期設定です。ここでは `日常の中で勝手に起きる身体イベント` をキャリブレーション材料にします。

カフレス血圧の高さ変化自己校正とは別に、より広い身体情報センシングへ使います。例えば呼吸推定、姿勢推定、装着位置推定、活動分類の個人化です。

### 例

- 立ち上がり: 重力方向と姿勢遷移の基準になる
- 階段: 周期的な上下動で歩行モデルを個人化できる
- 深呼吸: 呼吸由来の胸部/腹部IMU振動の基準になる
- タイピング開始: 腕時計PPGの低品質区間を予測する基準になる

### 評価

| 軸 | 見込み |
| --- | --- |
| 課題解決 | 中-高: キャリブレーション負担を下げる |
| 研究評価 | 高: プロトコル設計として面白い |
| 実用性 | 中: 対象タスクを絞れば可能 |
| 独自性 | 中-高: 対象イベント選びで差が出る |

### 最初の実験

M5Stack を体幹または手首に置き、日常的な5イベントを記録します。各イベントの前後で、姿勢推定や活動分類の安定性が上がるかを見るだけでも始められます。

## 案5: センサ装着位置の自己申告をなくす研究

### 一言

ユーザに「手首に装着してください」「胸に固定してください」と頼まず、センサ自身が装着位置と向きを推定してから測定モードを切り替える。

### 研究の転換

活動認識や健康推定は、装着位置が変わると性能が落ちます。そこを `装着位置を固定する` ではなく、`装着位置のズレを前提にする` に変えます。

### 実験案

- M5Stack/スマホを、手首、上腕、胸ポケット、ズボンポケット、机上に置く。
- 短い共通動作、例えば `立つ、座る、腕を上げる、歩く` を行う。
- センサ位置・向きを推定し、以後の特徴量を補正する。

### 評価

| 軸 | 見込み |
| --- | --- |
| 課題解決 | 高: 実用上よく起こる問題 |
| 研究評価 | 中-高: HARの一般化問題に接続 |
| 実用性 | 高: スマホ/ウェアラブルで効く |
| 独自性 | 中: 既存はあるが、簡単な再現性ある実験にしやすい |

## 案6: 呼吸を「値」ではなく「相互作用のリズム」として扱う

### 一言

呼吸数推定だけを目的にせず、ユーザの行動や緊張、作業切り替えに同期する身体リズムとして測る。

### 研究の転換

呼吸は健康指標として扱われがちですが、HCI 的には `作業と身体リズムの相互作用` として扱えます。例えば、集中作業、通知、会話、運動前後で呼吸の乱れ方がどう変わるかを測ります。

### 実験案

- M5Stack/スマホIMUを胸部または腹部に置き、呼吸由来の微小運動を測る。
- PC作業、通知割り込み、暗算、動画視聴、深呼吸誘導を行う。
- 呼吸数ではなく、呼吸周期の揺らぎ、停止、再同期までの時間を見る。

### 評価

| 軸 | 見込み |
| --- | --- |
| 課題解決 | 中: 明確な困りごとは後付けしやすい |
| 研究評価 | 高: CHI向きの問いにできる |
| 実用性 | 中: ストレス・作業支援へ展開可能 |
| 独自性 | 中-高: 値推定から相互作用へずらす点が良い |

## 案7: 「通知に気づいたか」を身体センサで推定する

### 一言

通知が見えたか、邪魔だったか、気づいたのに無視したかを、視線なしでもIMUや操作ログから推定する。

### 研究の転換

通知研究は、表示方法やタイミングを評価しがちです。ここでは `通知後の身体の反応` を測り、ユーザが明示的に答えなくても気づきや負担を推定します。

### M5Stackでの小実験

- 手首または机上にIMUを置く。
- PC作業中に通知を出す。
- 通知後1-5秒の手首微動、姿勢停止、タイピング停止、スマホ/マウス操作を記録する。
- 通知に気づいた/気づかない/邪魔だったを自己申告で取る。

### 評価

| 軸 | 見込み |
| --- | --- |
| 課題解決 | 中: 通知最適化へ接続 |
| 研究評価 | 高: CHIらしい |
| 実用性 | 中-高: 作業支援、AR、スマホ通知へ展開 |
| 独自性 | 中: 既存研究はあるが、低コストIMUに落とす余地 |

## 案8: センサデータを本人が訂正できる「身体ログ編集」UI

### 一言

活動認識の誤分類を、ユーザがあとから自然に直せるログ編集UIを作る。

### 研究の転換

センサモデルの性能を上げるだけでなく、間違った推定結果をユーザがどう見て、どこまで訂正したいかを研究対象にします。

### 実験案

- IMUで1日の簡易活動ログを作る。
- `歩行`, `座位`, `机作業`, `移動`, `不明` くらいに分類する。
- ユーザにタイムラインを見せ、間違いを訂正してもらう。
- 訂正負担、信頼、次回モデル改善への納得感を見る。

### 評価

| 軸 | 見込み |
| --- | --- |
| 課題解決 | 中-高: 実運用で必ず出る誤分類問題 |
| 研究評価 | 高: Human-AI interaction と接続 |
| 実用性 | 高: アプリ化しやすい |
| 独自性 | 中: UI設計で勝負 |

## 案9: 同じ身体データから「何を推定されたくないか」を調べる

### 一言

IMUや生体センサから推定できる情報をユーザに見せ、許容できる推定粒度を調べるプライバシー研究。

### 研究の転換

センサは「何が推定できるか」が語られがちですが、ユーザから見ると「何まで推定されると嫌か」が重要です。健康・行動・感情・生活リズムの境界を調べます。

### 実験案

- 同じIMUデータから、粗い推定と細かい推定を作る。
  - 粗い: 動いている/止まっている
  - 中: 歩行/座位/作業
  - 細かい: 生活パターン、緊張、疲労推定
- 各粒度について、保存・共有・研究利用の許容度を聞く。

### 評価

| 軸 | 見込み |
| --- | --- |
| 課題解決 | 中: 実用直結ではないが重要 |
| 研究評価 | 高: CHI/HCI向き |
| 実用性 | 中: 同意UIや研究倫理へ展開 |
| 独自性 | 中-高: デバイス計測寄りに作ると良い |

## 案10: センサを1つだけ使う「移動式身体プローブ」

### 一言

多数のセンサを常時装着するのではなく、1つのセンサを複数部位に短時間ずつ当てて身体状態を探索する。

### 研究の転換

ウェアラブルは「常時装着」が前提になりがちです。ここでは、測定を `短い探索行為` として設計します。

### 例

- 胸に10秒: 呼吸
- 手首に10秒: 動作/脈波補助
- 太ももに10秒: 姿勢/歩行
- 机上に10秒: 手作業振動

### 評価

| 軸 | 見込み |
| --- | --- |
| 課題解決 | 中: 常時装着が嫌な人向け |
| 研究評価 | 中-高: 測定行為のデザインとして面白い |
| 実用性 | 中: 習慣化できるかが鍵 |
| 独自性 | 高: 常時センシングからの転換 |

## すぐ着手するなら

### 1週間プロトタイプ

最も早いのは **案1 Signal Confidence Mirror** と **案2 Uncertainty-triggered micro-EMA** の組み合わせです。

1. M5Stack Core2 の6軸IMUを Bluetooth Classic で100Hz送信する。
2. Python側で1秒窓の特徴量を作る。
   - 加速度ノルム平均
   - 加速度ノルム分散
   - 角速度ノルム平均
   - 静止度
   - 周期性
3. `stable`, `moving`, `transition`, `unknown` の4状態を出す。
4. `unknown` または `transition` が続いた時だけ、PC画面に1問出す。
5. その応答によってモデルとログを更新する。

この形なら、医学的な正解値なしで研究として成立しやすいです。血圧や疾患推定のような高リスク領域に入らず、計測・HCI・人間情報工学として攻められます。

### 最初に作る成果物

- `sensor_log.csv`: 100Hz IMU
- `window_features.csv`: 1秒窓特徴
- `events.csv`: transition / unknown / stable
- `prompts.csv`: いつ質問したか、ユーザが答えたか
- `labels.csv`: ユーザ応答
- 可視化: 1日のタイムライン、質問タイミング、信号品質

## 私なら優先する順番

| 順位 | 案 | 理由 |
| --- | --- | --- |
| 1 | Signal Confidence Mirror | 既存装置で始められ、問いの置き方が良い |
| 2 | Uncertainty-triggered micro-EMA | CHI/UbiComp の流れに乗りやすく、実験設計も可能 |
| 3 | 測定不能性の地図 | 地味だが強い。論文化時に説得力が出る |
| 4 | 日常動作キャリブレーション | 実用性が高い。対象を絞ると化ける |
| 5 | 通知に気づいたか推定 | CHI寄り。デバイス計測から少し広げられる |

## 参考にした研究の流れ

- Multimodal micro-EMA on a smartwatch, CHI 2025: 20人7日間、5分ごとの姿勢/活動ラベル入力で、72.4%の応答率を報告。高頻度ラベリングの実現可能性と文脈バイアスの論点が参考になる。  
  https://veronikapotter.github.io/assets/pdfs/Le_MMuEMA_CHI_2025.pdf

- Sensing Noticeability in Ambient Information Environments, CHI 2025: 通知に気づいたかを、視線・頭部位置・操作・エンゲージメントから推定する方向。明示フィードバックに頼らないHCI計測の参考になる。  
  https://augmented-perception.org/publications/2025-sensing_noticeability.html

- SIG PhysioCHI, CHI EA 2025: physiological computing では技術的課題だけでなく、再現性、透明性、倫理、標準化が重要課題になっている。  
  https://research.monash.edu/en/publications/sig-physiochi-human-centered-physiological-computing-in-practice/

- AI and wearable IMUs in medicine systematic review, JMIR mHealth and uHealth 2025: IMU医療応用では参加者数中央値が小さく、実環境データ・ラベル・一般化が課題。  
  https://mhealth.jmir.org/2025/1/e60521/

- Generalizable HAR survey, arXiv 2025: IMU活動認識は、ユーザ差、装着位置差、環境差によるドメインシフトが大きな実用障壁。  
  https://arxiv.org/abs/2508.12213

- GenAI4HS @ UbiComp 2025: human sensing における基盤モデル、LLM、データ生成、in-the-wild deployment が議論対象。完璧な完成研究でなく、思考を促す初期探索も価値がある。  
  https://genai4hs.github.io/

- Smartphone IMU for cardiorespiratory condition screening, arXiv 2024: スマホIMUだけでも呼吸運動の取得・推定に使える可能性がある。  
  https://arxiv.org/abs/2408.15357

- Multimodal upper-arm physiological sensing, Biosensors 2025: PPG単独は皮膚色、動作、圧などの影響を受け、複数モダリティと信号品質判断が有効。  
  https://www.mdpi.com/2079-6374/15/8/493

## 採用判断メモ

今回は `独自性だけで勝つ` より、次のような案を優先すると良いです。

- 小さく実装できる
- 失敗してもデータ品質やユーザ行動の知見が残る
- 医学的な真値を必要としない
- 既存の6軸IMU/Bluetooth Classic環境を使える
- HCI的な問いと計測工学的な問いの両方を持てる

その意味で、`測定値の推定` よりも `測定が成立する条件を人とシステムで作る` 方向が、今の段階ではかなり良いと思います。
